# 프로젝트 계획서

## 1. 프로젝트 주제

내가 만들 프로그램의 주제:

스타트업 투자 유치 자격 진단기

---

## 2. 문제 상황

이 프로그램이 해결하려는 문제를 설명하세요.

내 설명:프로그램에서 매출액을 입력하라고 했는데, 사용자가 실수로 문자열("천만원")을 입력하거나 아무것도 입력하지 않고 엔터를 누르는 경우이다


---

## 3. 입력-처리-출력 구조

### 입력

사용자가 입력하거나 프로그램이 사용할 데이터는 무엇인가?

- 각 투자사의 이름, 최소 매출액 기준, 최소 특허 보유 개수 기준, 최소 팀원 수 기준이 포함된 2차원 리스트 데이터
- 진단받고자 하는 가상 스타트업의 연간 총 매출액 (단위: 만 원, 정수형 숫자)
- 스타트업이 현재 보유한 기술 특허의 개수 (단위: 개, 정수형 숫자)

### 처리

입력된 데이터를 어떤 규칙이나 알고리즘으로 처리할 것인가?

- 사용자로부터 매출액, 특허 수, 팀원 수를 입력받고, 숫자가 아닌 값이 입력되거나 음수가 입력되는 오류 상황을 조건문으로 방지함.
- 투자사 데이터가 저장된 2차원 리스트를 for 반복문으로 한 줄씩 순회하며, 사용자가 입력한 스타트업의 수치가 투자사의 최소 기준들보다 크거나 같은지(>=) 비교함.
- 조건문(if)을 활용해 3가지 기준(매출, 특허, 팀원)을 모두 만족하는 투자사만 '신청 가능 리스트'에 따로 저장하고, 만족하지 못하는 경우 부족한 항목을 분석함.

### 출력

프로그램이 최종적으로 무엇을 출력하는가?

- 사용자가 입력한 스타트업의 정보(매출, 특허, 팀원 수)를 깔끔하게 요약하여 먼저 보여줌.
- 기준을 통과하여 최종적으로 투자 유치 신청이 가능한 투자사들의 이름과 해당 투자사의 조건 정보 목록.

---

## 4. 2차원 리스트 설계

내 프로그램에서 사용할 2차원 리스트의 이름:

# 투자사 데이터 구조 안내
# 각 행(세로)은 하나의 투자사를 나타내고, 
# 각 열(가로)은 [투자사 이름, 최소 매출액 기준(만원), 최소 특허 수 기준(개), 최소 팀원 수 기준(명)]을 의미해.

investors = [
    ["A앤젤스", 1000, 0, 2],
    ["B벤처스", 5000, 1, 3],
    ["C인베스트", 15000, 2, 5],
    ["D파트너스", 50000, 3, 10]
]
```

### 한 행의 의미
"하나의 특정 투자사가 가지고 있는 모든 기준 정보 세트(묶음)"

열 번호의미예시0번 열투자사 이름 (텍스트 형태)
1번 열투자사에서 요구하는 최소 매출액 기준 (단위: 만 원)
2번 열투자사에서 요구하는 최소 기술 특허 보유 개수 (단위: 개)
3번 열투자사에서 요구하는 최소 팀원(직원) 수 기준 (단위: 명)

## 5. 함수 설계

함수 이름역할입력값반환값 또는 출력input_startup_data사용자로부터 매출액, 특허 수, 팀원 수를 안전하게 입력받음 (음수 입력 시 재입력 요구하는 예외 처리 포함)없음 (사용자 키보드 입력)[매출액, 특허수, 팀원수] 형태의 1차원 리스트check_investment_eligibility2차원 리스트를 순회하며 사용자의 지표와 투자사별 최소 자격 조건을 비교하여 통과 여부를 판정함investor_list (투자사 2차원 리스트),startup_data (기업 지표 1차원 리스트)자격 조건을 모두 만족하여 통과된 투자사들만 모은 2차원 리스트print_diagnostic_report입력받은 기업 정보 요약과 최종 통과된 투자사 목록, 그리고 맞춤형 조언 피드백을 화면에 출력함startup_data (기업 지표 1차원 리스트),eligible_list (통과된 투자사 리스트)진단 결과 보고서 화면 (콘솔 텍스트 출력)

## 6. 사용할 제어구조

### 조건문을 사용할 부분

# 만약 스타트업의 매출액이 투자사 기준보다 크거나 같고, 
# 특허 수도 기준보다 크거나 같고, 팀원 수도 기준보다 크거나 같다면?
if 매출액 >= 투자사_최소매출 and 특허수 >= 투자사_최소특허 and 팀원수 >= 투자사_최소팀원:
    # 이 투자사는 신청 가능 목록에 추가!
else:
    # 어떤 부분이 부족한지 체크해서 피드백 준비!

내 계획:
특정 투자사의 기준 데이터와 사용자가 입력한 스타트업의 정보(매출액, 특허 수, 팀원 수)를 하나씩 직접 비교할 때 사용함. 매출액, 특허 수, 팀원 수가 모두 투자사의 최소 기준 이상이어야 하므로 and 연산자로 묶어서 판정함.
(예시: 만약 매출액이 A앤젤스 기준 이상이고, 특허 수도 기준 이상이고, 팀원 수도 기준 이상이라면 'A앤젤스 투자 자격 통과'라고 판정함)

### 반복문을 사용할 부분

예) 2차원 리스트의 모든 항목을 하나씩 검사한다.

내 계획:2차원 리스트 investors에 저장된 모든 투자사의 데이터를 처음부터 끝까지 한 행씩 차례대로 꺼내어 확인(순회)할 때 사용함. 투자사가 여러 개 있어도 일일이 코드를 복사할 필요 없이, 반복문을 통해 모든 투자사를 빠짐없이 검사함.

## 7. 예상되는 예외 상황

예) 조건에 맞는 추천 결과가 없을 때, 잘못된 입력을 했을 때 등

1. 사용자가 숫자가 아닌 문자열이나 기호를 입력하는 상황
이유: 매출액, 특허 수, 팀원 수를 입력받을 때 사용자가 실수로 한글("천만원")이나 영문, 기호를 입력하면 파이썬의 int() 변환 과정에서 프로그램이 강제로 멈추는 오류(ValueError)가 발생할 수 있습니다.
2. 입력값에 음수(0보다 작은 수)를 입력하는 상황
이유: 경영 데이터의 특성상 매출액이 -500만 원이 되거나 팀원 수가 -2명이 되는 것은 논리적으로 불가능합니다.
