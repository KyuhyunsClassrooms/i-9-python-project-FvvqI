## 1. 프로그램 주제와 목적

내 프로그램의 주제:

이 프로그램을 만든 이유:스타트업 투자 유치 자격 진단기


---

## 2. 문제 분해 및 모델링

### 입력 - 각 투자사의 이름, 최소 요구 매출액(단위: 만 원), 최소 기술 특허 보유 개수(단위: 개), 최소 상시 팀원 수(단위: 명) 데이터


### 처리- 사용자로부터 데이터를 입력받을 때, 매출액, 특허 수, 팀원 수에 음수(0보다 작은 값)가 입력되면 조건문(if)을 통해 경고를 띄우고, 올바른 값이 들어올 때까지 반복문(while)을 통해 재입력을 요구함.


### 출력
기준을 통과한 투자사들의 이름과 해당 투자사의 상세 자격 요건(최소 지표들) 목록을 깔끔한 형식으로 출력함.

---

## 3. 사용한 2차원 리스트 설명

2차원 리스트 이름:investors

한 행의 의미:프로그램에 등록된 '특정 가상 투자사 1곳의 모든 투자 자격 조건 데이터 세트(묶음)'를 의미합니다. (예: investors[0]은 'A앤젤스' 투자사가 요구하는 조건들의 집합)

각 열의 의미:

| 열 번호 | 의미 | 실제 예시 |
|---:|---|---|
| 0번 열 |투자사를 식별하는 투자사 이름  | "A앤젤스", "B벤처스" |
| 1번 열 | 투자사에서 요구하는 최소 연간 매출액 기준 | 1000 (1000만 원 이상 필요) |
| 2번 열 | 투자사에서 요구하는 최소 기술 특허 보유 개수 |0 (특허 무관), 1 (1개 이상 필요)  |
| 3번 열 | 투자사에서 요구하는 최소 상시 팀원(직원) 수 기준  | 2 (2명 이상), 3 (3명 이상 필요) |

이 2차원 리스트를 사용한 이유:

투자사 한 곳의 정보(이름, 매출, 특허, 팀원)를 흩어지지 않게 한 행으로 깔끔하게 묶어서 관리하려고 썼다
투자사가 4개든 100개든 코드 여러 번 복사할 필요 없이, for 반복문 단 한 줄로 모두 검사하려고 썼어.
나중에 새로운 투자사가 생겨도 코드를 고칠 필요 없이 리스트에 한 줄만 슥 추가하면 프로그램이 알아서 계산해 주기 때문이야.

## 4. 함수 설명

| 함수 이름 | 역할 | 입력값 | 반환값 또는 출력 |
|---|---|---|---|
|  input_startup_data|  사용자로부터 매출액, 특허 수, 팀원 수를 안전하게 입력받음 (음수 예외 처리 포함)|없음 (사용자 키보드 입력)  |[매출액, 특허수, 팀원수] 형태의 1차원 리스트  |
| check_investment_eligibility |2차원 리스트를 순회하며 스타트업 지표와 투자사 조건을 비교 및 필터링 | investor_list (투자사 2차원 리스트),
startup_data (내 기업 1차원 리스트) | 자격 조건을 만족한 투자사들만 모은 2차원 리스트 |
|print_diagnostic_report  | 입력받은 스타트업의 정보 요약과 최종 통과된 투자사 목록 및 맞춤 조언을 화면에 출력 | startup_data (내 기업 1차원 리스트),
eligible_list (통과된 투자사 리스트) | 진단 결과 보고서 화면 (콘솔 텍스트 출력) |

가장 중요한 함수 하나를 골라 자세히 설명하세요.

함수 이름:check_investment_eligibility

설명:이 함수는 프로그램의 가장 핵심이 되는 '투자 자격 판정 알고리즘'을 수행하는 함수입니다. 전체 투자사 데이터가 저장된 2차원 리스트와 사용자가 입력한 스타트업의 지표 리스트를 전달받아 연산을 시작합니다. 함수 내부에서는 for 반복문을 사용하여 2차원 리스트의 각 행(개별 투자사 데이터)을 처음부터 끝까지 순서대로 꺼내옵니다. 그다음 if 조건문과 and 논리 연산자를 사용하여 사용자의 '매출액, 특허 수, 팀원 수'가 해당 투자사의 '최소 요구 기준'보다 모두 크거나 같은지(>=) 동시에 비교합니다. 3가지 조건을 모두 만족하여 자격 검증을 통과한 투자사 데이터(행)만 따로 빈 리스트(eligible_investors)에 .append() 함수로 누적하여 모은 뒤, 최종 필터링된 리스트를 반환합니다. 이 함수 덕분에 투자 데이터가 아무리 많아져도 코드를 중복해서 쓰지 않고 일괄적이고 효율적으로 조건을 연산할 수 있습니다.
